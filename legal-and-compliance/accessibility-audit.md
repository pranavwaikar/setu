# Accessibility Audit Report (WCAG Compliance)

**Last Updated: June 2026**

This document details the accessibility standards and review status for the Setu project dashboard and documentation pages.

---

## 1. Compliance Target
Setu aims to design user interfaces that are accessible to all users, adhering to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA recommendations where feasible.

## 2. Implemented Features
*   **Semantic HTML:** We use native HTML5 structures (`<main>`, `<header>`, `<nav>`, `<button>`, `<input>`) to ensure screen readers can accurately interpret the layout.
*   **Keyboard Navigation:** All dashboard elements and interactive tabs support full keyboard focus and activation without requiring mouse interaction.
*   **Contrast Ratios:** The theme utilizes text colors and element borders that meet or exceed contrast ratio guidelines against dark background elements.
*   **Labels and ARIA Roles:** Text fields, input checkboxes, and button controls include descriptive text labels or custom `aria-label` tags.

## 3. Continuous Improvement
Accessibility is an ongoing process. If you encounter any barriers or accessibility bugs while navigating the Setu dashboard, please file a GitHub issue detailing the browser, assistive technology, and steps to reproduce.
